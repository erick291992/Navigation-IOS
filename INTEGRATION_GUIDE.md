# UniversalMediaPicker Integration Guide 💎⚓

This guide provides the architectural and implementation details required to integrate the **UniversalMediaPicker (V3)** suite into any SwiftUI project.

## 1. Modular Requirements

The picker is designed to be zero-config for basic use, but relies on the following Apple frameworks:
- `SwiftUI` & `Observation`
- `Photos` & `PhotosUI`
- `AVFoundation`

### Mandatory Info.plist Keys
You **must** add these keys to your target's `Info.plist` to prevent runtime crashes:
```xml
<key>NSPhotoLibraryUsageDescription</key>
<string>We need access to your library to select photos and videos.</string>
<key>NSCameraUsageDescription</key>
<string>We need camera access to take new photos and videos.</string>
<key>NSMicrophoneUsageDescription</key>
<string>We need microphone access for video recording.</string>
```

## 2. Basic Usage (The Gold Standard)

The easiest way to trigger the picker is via the `.mediaPicker` ViewModifier.

```swift
import SwiftUI

struct ContentView: View {
    @State private var isPickerPresented = false
    @State private var selectedItems: [MediaItem] = []

    var body: some View {
        VStack {
            Button("Open Picker") {
                isPickerPresented = true
            }
        }
        .mediaPicker(
            isPresented: $isPickerPresented,
            configuration: .init(
                crop: .square,
                selectionLimit: 1
            ),
            onCompletion: { items in
                self.selectedItems = items
                print("Successfully picked \(items.count) items.")
            }
        )
    }
}
```

## 3. Configuration & Styling

The `MediaPickerConfiguration` object allows for deep tailoring of the experience.

### Configuration Properties
| Property | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| `selectionLimit` | `Int` | `1` | Total items a user can select. |
| `crop` | `MediaCrop` | `.freeform` | Enforced crop aspect ratio (`.square`, `.freeform`, `.circle`). |
| `style` | `PickerStyle` | `.pinkSleek` | The visual theme (colors, corner radius, animations). |

### Custom Styling Example
```swift
let customStyle = PickerStyle(
    accentColor: .blue,
    backgroundColor: .black,
    cornerRadius: 12,
    fontDesign: .rounded
)

let config = MediaPickerConfiguration(style: customStyle)
```

## 4. Understanding the `MediaItem`

The picker returns an array of `MediaItem` objects. This is a unified model that abstracts away the complexity of `PHAsset`, `UIImage`, and temporary file URLs.

```swift
public struct MediaItem: Identifiable {
    public let id: UUID
    public let type: MediaType         // .photo or .video
    public let thumbnail: UIImage      // High-performance preview
    public let originalURL: URL?       // Path to full-res file
    public let asset: PHAsset?         // Original PhotoKit asset (if applicable)
}
```

## 5. Advanced: Headless Engine Usage

If you want to build a custom UI but use our high-performance processing engine:

```swift
let engine = MediaPickerEngine.shared

// Process PhotosPickerItems
let items = try await engine.process(photosPickerItems)

// Process raw UIImages
let item = try await engine.process(myUIImage)
```

## 6. Troubleshooting

- **Double-Sheet Flickering**: Always use the `.mediaPicker` modifier on the **highest possible parent view** (usually the `NavigationView` or `ZStack` root) to ensure the cropping layer pushes correctly.
- **Permission Denied**: The picker includes built-in `PermissionNeededView`. If a user denies access, they will see a "Enable Access" button that links directly to Settings.
